# Отчёт о функциональности Middleware сервиса
## Для заказчика (не технический язык)

---

## Что это за сервис?

Это автоматический посредник между вашим интернет-магазином (Shopify) и системой управления продажами (Bitrix24). Сервис работает как переводчик и помощник, который:

- **Автоматически передаёт заказы** из интернет-магазина в систему управления
- **Синхронизирует данные** между двумя системами в обе стороны
- **Управляет складскими остатками** товаров
- **Отслеживает статусы заказов** и обновляет их автоматически

---

## Что сервис уже умеет делать?

### 1. 📦 Автоматическая синхронизация заказов

**Из Shopify → Bitrix24:**
- Когда клиент оформляет заказ на сайте, сервис автоматически создаёт сделку в Bitrix24
- Все данные о заказе (товары, цены, покупатель, адрес доставки) переносятся автоматически
- Заказ попадает в правильную воронку продаж в зависимости от того, где он был оформлен:
  - Заказы из магазина (POS) → воронка "Stock (in the shop)"
  - Заказы с сайта → воронка "Stock (site)"
  - Предзаказы → воронка "Pre-order"

**Из Bitrix24 → Shopify:**
- Когда менеджер создаёт сделку в Bitrix24 с товарами, сервис автоматически создаёт технический заказ в Shopify
- Это нужно для правильного учёта товаров на складе

### 2. 🔄 Автоматическое обновление статусов

**Статусы оплаты:**
- Если заказ оплачен в Shopify → статус в Bitrix24 меняется на "Оплачен"
- Если заказ отменён → статус меняется на "Отменён"
- Если сделан возврат → статус обновляется соответственно

**Статусы выполнения (Fulfillment):**
- Когда заказ отправлен клиенту в Shopify → статус в Bitrix24 обновляется
- Отслеживается полная отправка или частичная

### 3. 📊 Управление товарами и складом

**Синхронизация остатков:**
- Сервис сравнивает количество товаров в Shopify и Bitrix24
- Если количества не совпадают, автоматически создаются документы прихода или списания
- Это гарантирует, что остатки всегда актуальны в обеих системах

**Создание товаров:**
- Можно массово создавать товары в Bitrix24 из каталога Shopify
- Товары автоматически распределяются по категориям (A-F, G-M, N-S, T-Z)
- Заполняются все необходимые поля: размер, цвет, бренд, категория

**Сертификаты:**
- Специальная обработка подарочных сертификатов
- Автоматическое создание и обновление сертификатов в Bitrix24

### 4. 👤 Управление контактами

- При создании заказа автоматически создаётся или обновляется контакт покупателя в Bitrix24
- Сохраняются все данные: имя, телефон, email, адрес доставки

### 5. 💰 Обработка возвратов и отмен

- Если заказ возвращён в Shopify, сервис автоматически обновляет сделку в Bitrix24
- Обрабатываются как полные, так и частичные возвраты
- Статус сделки меняется соответственно

### 6. 🏪 Специальная обработка POS-заказов

- Заказы, оформленные в физическом магазине через кассу (POS), автоматически попадают в правильную воронку
- Отличаются от заказов с сайта для правильной отчётности

### 7. 📝 Технические заказы

- Когда менеджер создаёт сделку в Bitrix24, но заказа в Shopify ещё нет, сервис автоматически создаёт технический заказ
- Такие заказы помечаются специальными метками (TECH, BITRIX) для идентификации

### 8. 🔍 Мониторинг и логирование

- Все операции записываются в логи
- Можно отследить любую синхронизацию и найти проблемы
- Есть веб-интерфейс для просмотра всех событий и операций

## Веб-интерфейс (UI)

Сервис имеет удобный веб-интерфейс, где можно:

### Просмотр событий
- Видеть все заказы, которые пришли из Shopify
- Видеть все события, которые пришли из Bitrix24
- Просматривать детали каждого события

### Ручная отправка
- Если автоматическая синхронизация не сработала, можно вручную отправить заказ из Shopify в Bitrix24
- Или наоборот, отправить событие из Bitrix24 в Shopify

### Управление товарами
- Массовое создание товаров по категориям
- Синхронизация остатков товаров
- Обновление информации о товарах

### Логи и отчёты
- Скачать все логи для анализа
- Просмотреть историю всех операций
- Найти ошибки и проблемы

✅ **Прозрачность** - все операции видны в логах

## Что происходит автоматически?

1. **Клиент оформляет заказ на сайте** → Заказ появляется в Bitrix24 через несколько секунд
2. **Менеджер меняет статус в Bitrix24** → Статус обновляется в Shopify
3. **Заказ отправлен клиенту** → Статус выполнения обновляется в Bitrix24
4. **Товар закончился на складе** → Остатки синхронизируются автоматически
5. **Создана сделка в Bitrix24** → Создаётся технический заказ в Shopify

## Безопасность

- Все операции защищены токенами доступа
- Проверяется подлинность данных от Shopify и Bitrix24
- Защита от дублирования операций (каждая операция выполняется только один раз)
- Все действия логируются для аудита

## Техническая поддержка

Сервис работает 24/7 и автоматически обрабатывает все события. При возникновении проблем:

- Можно просмотреть детали любой операции

## Известные ограничения и проблемы

### ⚠️ Инвентаризация товаров

**Текущее состояние:**
- На данный момент система обрабатывает только товары, которые имеют количество на складе больше нуля (`qty > 0`)
- Товары с нулевым остатком не синхронизируются автоматически
- Это сделано для оптимизации процесса и избежания обработки неактуальных товаров

**Что это означает:**
- Если товар закончился на складе (количество = 0), он не будет автоматически обновляться в Bitrix24
- Для полной синхронизации всех товаров (включая с нулевым остатком) требуется дополнительная настройка

### ⚠️ Обновление полей товаров

**Известная проблема:**
- При обновлении товаров в Bitrix24 возникают сложности с обновлением некоторых полей:
  - Поле **Size** (Размер) - не всегда обновляется корректно
  - Другие пользовательские поля могут требовать ручного обновления

**Временное решение:**
- Для критически важных товаров рекомендуется ручное обновление полей через интерфейс Bitrix24
- При массовом обновлении товаров некоторые поля могут не синхронизироваться автоматически

**Планируется исправить:**
- Улучшение логики обновления полей товаров
- Более надёжная синхронизация всех полей, включая Size и другие пользовательские свойства
